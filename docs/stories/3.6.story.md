## Acceptance Criteria

**Functional Requirements:**

1. `ActionItemList` component renders inside `SessionCard` expanded view (replacing TODO at line 157)
2. Each action item row shows: checkbox, description text, assignee badge (Coach=blue / Client=green), status dropdown
3. Checkbox click ‚Üí `PUT /api/action-items/[id]` `{ status: 'completed', completed_at: now }` ‚Üí strikethrough + `opacity-50`

6. Enter key submits Add Action Item form
7. Pending action items count on Client Detail header updates reactively when items are completed

**Integration Requirements:**

8. `PendingActionsSection` (Client Detail page) continues to work unchanged

10. New functionality follows React Query optimistic update pattern from `PendingActionsSection`
11. `['action-items', clientId]` query invalidated on mutation so Client Detail header count updates

**Quality Requirements:**

12. Mobile-friendly: checkboxes and dropdowns usable on touch
13. No regression in `PendingActionsSection`, `SessionCard`, or Client Detail stats
14. Tests cover: render list, checkbox complete, status change, add item form submit
