# Story 3.6: Action Item Tracking UI

## Status
Not Started

## Story

**As a** executive coach,
**I want** to see, manage, and complete action items from coaching sessions,
**so that** I can track commitments made by both me and my clients without relying on memory.

## Acceptance Criteria

1. `ActionItemList` component displays within expanded SessionCard
2. Each action item shows: description text, assignee badge (Coach / Client), status dropdown
3. Checkbox marks item complete → `status = 'completed'`, `completed_at` recorded
4. Status dropdown options: Pending | In Progress | Completed | Cancelled
5. Completed items shown with strikethrough text and dimmed appearance
6. "Add Action Item" button (manual creation) — inline input within the list
7. Manual action item creation: description text input + assignee selector (Coach/Client) → `POST /api/action-items`
8. Pending action items count shown on Client Detail header (updates reactively)
9. Action items also visible in Pending Actions section on Client Detail page (Story 2.6)
10. Mobile friendly: checkboxes and status dropdowns usable on touch

## Component to Create

### `apps/web/src/components/sessions/ActionItemList.tsx`
Props: `{ sessionId: string; clientId: string; items: ActionItem[] }`
- List of action item rows
- Each row: checkbox, description, assignee badge, status dropdown
- "Add Action Item" form at bottom
- Optimistic updates for checkbox/status changes

## Tasks / Subtasks

- [ ] **ActionItemList Component**
  - [ ] Render list of action items (from props or React Query)
  - [ ] Each item row: checkbox (quick-complete) + description + assignee badge + status select
  - [ ] Checkbox: click → PUT `/api/action-items/[id]` with `{ status: 'completed', completed_at: now }`
  - [ ] Status dropdown: onChange → PUT `/api/action-items/[id]` with `{ status }`
  - [ ] Completed items: strikethrough + opacity-50
  - [ ] Assignee badges: "Coach" (blue badge) or "Client" (green badge)

- [ ] **Add Action Item Form**
  - [ ] Text input for description (required)
  - [ ] Assignee select: Coach | Client
  - [ ] Submit → POST `/api/action-items` → append to list
  - [ ] Keyboard shortcut: Enter to submit

- [ ] **Optimistic Updates**
  - [ ] Mark complete: show as completed immediately, revert on API error
  - [ ] Status change: apply immediately, revert on API error
  - [ ] Use React Query `useMutation` with `onMutate` for optimism

- [ ] **Integration: Client Detail Header**
  - [ ] Count pending action items across all sessions for this client
  - [ ] Display count in stats row: "5 Pending Actions"
  - [ ] Update count when items are completed

- [ ] **Integration: SessionCard Expanded View**
  - [ ] Render ActionItemList inside expanded SessionCard
  - [ ] Pass session's action_items as props (already fetched with session data)

## Dev Notes

- Action items for a session can be loaded with the session detail or fetched separately
- Use shadcn/ui Checkbox, Select components
- Assignee badge: use shadcn/ui Badge with variant="outline" for Coach (blue text) and variant="outline" for Client (green text)
- For the Pending Actions section on Client Detail (Story 2.6): fetch with `GET /api/action-items?client_id=[id]&status=pending`
