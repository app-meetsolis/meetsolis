# Story 6.4: Coach Brief Screen (Hero Feature)

## Status
Ready for Dev

## Locked Decisions (2026-05-11)
- **Hero feature framing:** This is THE moment coaches realize MeetSolis ≠ Otter.ai. Justifies $99/mo. Quality bar is highest in the product. No generic AI output — every brief must feel hand-written.
- **Activation trigger:** Background check every 5 min. Fire when `start_time - now() <= user_preferences.coach_brief_window_minutes` (default 60 min, user-configurable in Story 6.5).
- **Generation source:** Solis `hybrid_session_search` (Story 4.1) + 3 most recent sessions verbatim (always — never trust retrieval alone for recency). Prompt MUST pass 10-real-transcript test gate (BRAINSTORM §9).
- **Storage:** `coach_briefs` table, JSONB content field, one brief per `calendar_event_id` (unique constraint).
- **Tier gating:** Coach Brief screen + dashboard banner = Pro only. Free coaches CAN access prep via Solis chat (inline response only — no dedicated brief screen). This drives Coach Brief upgrade pressure organically.
- **Route:** `/brief/[eventId]` — full page, NOT modal. Coach needs focus.
- **Theme:** Dark deep teal/green production palette. NOT old PRD earth-tone palette.
- **Editability:** Every AI field is editable inline (click → edit → save on blur). AI is assistant, not authority.
- **Coach note field:** Private free-text field per client (persists across sessions, NOT per-brief). Stored on `clients.coach_notes`. Never AI-touched.
- **Solis natural language trigger:** Detect prep-intent in Solis chat ("prep me for [Name]", "prepare for [Name]'s session"). Branch: matched calendar event → CTA to brief screen. Unmatched → inline brief in chat. Works for BOTH Pro and Free (Free gets inline, Pro gets dedicated screen).
- **Multiple meetings same day:** Brief shows next event first. "Next brief →" button in header cycles forward only (linear). Button disappears after last brief of day.
- **First-ever session:** Zero history → placeholder "First session with [Name] — no history yet" + 3 suggested opening questions (AI-generated from client card goal/notes).
- **Dismissal:** Brief dismissed → `coach_briefs.dismissed = true`. "Show brief" button persists in dashboard for 2 hours post-meeting (recoverable).
- **PDF export:** STRETCH. Defer to follow-up if time tight. Print-friendly CSS only at minimum.
- **Countdown timer:** Updates every minute. "in 47 minutes" / "in 1 hour" / "starts now" / "started 5 min ago" (within 30 min after start).

## Story

**As a** Pro coach,
**I want** a pre-session brief to automatically appear before any matched calendar event,
**so that** I walk into every session fully prepared without spending any time on manual prep — and never forget what we covered last time.

### Brief Generation Logic
- [ ] In `apps/web/src/lib/brief/generate-brief.ts`:
  - Fetch client: `clients` row by `client_id`.
  - Fetch last session: `sessions` where `client_id = $1` ORDER BY `created_at DESC` LIMIT 1.

  - Fetch recent sessions verbatim: last 3 sessions by `created_at`.
  - Fetch top breakthroughs: top 3 sessions by relevance via `hybrid_session_search` with query `"breakthrough OR insight OR realization"` filtered to this client.
  - Build AI Prep Note via Claude (see prompt spec below).
  - Compose final JSONB content (see structure below).

### Solis Natural Language Trigger
- [ ] In `apps/web/src/lib/ai/solis.ts` query handler — add intent detection BEFORE main RAG call:

    - Look up nearest upcoming `calendar_events` for this client (within next 7 days).

```sql
CREATE TABLE coach_briefs (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES users(id) ON DELETE CASCADE,
```

### Solis Intent Detection
```ts
const PREP_INTENT_REGEX = /(prep me for|prepare me for|prep for|brief me on)\s+(.+?)(?:[.?!]|$)/i;

function detectPrepIntent(query: string, userClients: Client[]): { isPrep: boolean; matchedClient?: Client } {
  const match = query.match(PREP_INTENT_REGEX);
  if (!match) return { isPrep: false };
  const namePhrase = match[2].trim().toLowerCase();
  const matchedClient = userClients.find(c =>
    namePhrase.includes(c.name.split(' ')[0].toLowerCase()) ||
    namePhrase.includes(c.name.toLowerCase())
  );
  return { isPrep: true, matchedClient };
}
```
Edge case: ambiguous name → no match → fall through to generic Solis RAG.


### Countdown Format
```ts
function formatCountdown(startTime: Date): string {
  const diffMin = differenceInMinutes(startTime, new Date());
  if (diffMin > 60) return `in ${Math.floor(diffMin / 60)} hour${diffMin >= 120 ? 's' : ''}`;
  if (diffMin > 1) return `in ${diffMin} minutes`;
  if (diffMin >= -1) return 'starts now';
  if (diffMin >= -30) return `started ${Math.abs(diffMin)} min ago`;
  return 'session ended';
}
```
