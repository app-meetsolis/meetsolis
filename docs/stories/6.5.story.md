# Story 6.5: Coach Brief Settings, Bot Toggle & Live Transcript View

## Status
Ready for Dev

## Locked Decisions (2026-05-11)
- **Coach Brief activation window:** dropdown in Settings → Preferences. Options: 30 min / **1 hour (default)** / 2 hours / 4 hours. Stored as `user_preferences.coach_brief_window_minutes` (`30 | 60 | 120 | 240`).
- **Bot global toggle:** "Auto-transcribe sessions with MeetSolis Notetaker" — on/off. Default = `true` for Pro (already added in Story 6.2 migration). UI ships here.
- **Per-meeting bot skip:** "⋯" menu on Upcoming Sessions card row (Story 6.1). Sets `calendar_events.bot_skipped = true`.
- **Manual upload transcription provider preference:** dropdown — `Deepgram Nova-2` (default) / `Gladia`. Stored as `user_preferences.manual_transcription_provider`. **Separate from bot sessions** — bot always uses Gladia.
- **Live transcript visibility:** **HIDDEN by default**. Coach must explicitly click "Show live transcript ▾" to expand. Reason: reduces visual noise during session, coach is in the meeting, not staring at MeetSolis.
- **Live transcript update mechanism:** Supabase Realtime subscription on `recall_sessions.live_transcript` column. Recall streams transcript chunks → we append to JSONB array → Realtime pushes to client.
- **Live transcript auto-collapse:** panel auto-closes when `recall_sessions.status` transitions from `in_meeting` to `done`.
- **Speaker labels in live transcript:** use mapped names once Story 6.3 confirms mapping. Until then, raw `speaker_0` / `speaker_1` labels.
- **Tier gating:** Settings tab visible to all. Pro-only sections grayed out with "Upgrade for [feature]" inline CTA for Free coaches (rather than hiding entirely — drives upgrade awareness).

## Story

**As a** Pro coach,
**I want** to configure when my Coach Brief activates, toggle the auto-transcription bot, and optionally watch live transcripts mid-session,
**so that** MeetSolis matches my personal workflow instead of forcing me into defaults.

### Supabase Realtime Setup
- Enable Realtime on `recall_sessions` table (Supabase dashboard or migration via `ALTER PUBLICATION supabase_realtime ADD TABLE recall_sessions`).
- Client subscribes:
  ```ts
  supabase
    .channel(`recall_sessions:user_id=eq.${userId}`)
    .on('postgres_changes', { event: 'UPDATE', schema: 'public', table: 'recall_sessions' },
      payload => updateLiveTranscript(payload.new))
    .subscribe();
  ```
- RLS already covers per-user filtering.

### DB Migration
```sql
-- user_preferences extensions
ALTER TABLE user_preferences
  ADD COLUMN IF NOT EXISTS coach_brief_window_minutes integer NOT NULL DEFAULT 60
    CHECK (coach_brief_window_minutes IN (30, 60, 120, 240)),
  ADD COLUMN IF NOT EXISTS manual_transcription_provider text NOT NULL DEFAULT 'deepgram'
    CHECK (manual_transcription_provider IN ('deepgram', 'gladia'));

-- recall_sessions.live_transcript
ALTER TABLE recall_sessions
  ADD COLUMN IF NOT EXISTS live_transcript jsonb NOT NULL DEFAULT '[]'::jsonb;

-- Enable Realtime
ALTER PUBLICATION supabase_realtime ADD TABLE recall_sessions;
```

Update `packages/shared/src/types/UserPreferences.ts` accordingly.

### Live Transcript Performance
- Transcript chunks arrive ~every 2–5s during active session.
- 60-min session ≈ 720–1800 chunks.
- JSONB append on every chunk = OK at this scale (Postgres handles JSONB appends efficiently with `||` operator).
- Optimization (defer if needed): batch chunks every 10s before append.
