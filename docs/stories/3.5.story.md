# Story 3.5: Session Timeline UI

## Status
Not Started

## Story

**As a** executive coach,
**I want** to see all my sessions for a client in a clear reverse-chronological timeline,
**so that** I can quickly review session history and upload new sessions.

## Acceptance Criteria

1. Session timeline renders within Client Detail page (Story 2.6)
2. Sessions displayed in reverse-chronological order (newest first)
3. "Upload Session Transcript" button at top of timeline opens SessionUploadModal
4. Empty state shown when no sessions exist: "No sessions yet. Upload your first session transcript to get started." with upload button
5. Loading skeleton (3 skeleton cards) shown while sessions are fetching
6. **SessionCard (collapsed):** date, title, summary snippet (first 150 chars), key topic badges (max 3, truncated), action item count badge
7. **SessionCard (expanded):** full summary, all key topics, complete ActionItemList, "View Full Transcript" link
8. Sessions with `status = 'processing'` show animated spinner + "AI is processing your session..." text
9. Sessions with `status = 'error'` show error message + "Retry" button
10. SessionUploadModal has 2 tabs: "Manual Upload / Paste" (Story 3.2) and "Auto-Transcribe Audio" (Story 3.3)
11. Mobile responsive: cards stack vertically, expand/collapse works on touch

## Components to Create

### `apps/web/src/components/sessions/SessionTimeline.tsx`
Props: `{ clientId: string }`
- Fetches sessions via React Query (`GET /api/sessions?client_id=[clientId]`)
- Renders upload button + list of SessionCard components
- Handles empty state and loading skeleton

### `apps/web/src/components/sessions/SessionCard.tsx`
Props: `{ session: Session }`
- Collapsed/expanded toggle via local state
- Renders session summary, topics, action items (expanded)
- Shows processing/error states

### `apps/web/src/components/sessions/SessionUploadModal.tsx`
Props: `{ clientId: string; onSuccess: () => void }`
- shadcn/ui Dialog component
- Tabs: Manual Upload | Auto-Transcribe
- Tab 1: File input (.txt/.docx) + textarea paste + date + title
- Tab 2: File input (audio/video) + date + title
- Form validation and submission

## Dev Notes

- Use `date-fns` (already likely installed) for date formatting: "Feb 15, 2026"
- Key topic badges: use shadcn/ui Badge component with muted style
- Action item count: "3 actions" badge (green if all complete, amber if any pending)
- Refetch interval for processing sessions: use React Query's `refetchInterval` with conditional (only refetch if any session has status = 'processing')
