## Status

## Story

**As a** executive coach,
**I want** to add and edit free-form notes on a client record,
**so that** I can capture context, observations, and information that doesn't fit into session transcripts.

## Story Context

**Existing System Integration:**
- Integrates with: `PUT /api/clients/[id]`, `GET /api/clients/[id]`
- Technology: Next.js 14 App Router, React Query, Tailwind, Shadcn UI, sonner toast
- Follows pattern: existing ClientForm, DeleteClientDialog patterns
- Touch points: `packages/shared/src/schemas/client.ts` (ClientUpdateSchema already has `notes`), `clients.notes TEXT` column (migration 015)

**Dependency note:** Story 2.6 (full client detail view) was paused and never merged. This story creates a **minimal** `/clients/[id]` page stub sufficient to host the notes editor. Story 2.6 will expand this page later with session timeline, action items, etc.

## Acceptance Criteria

1. Client detail page exists at `/clients/[id]` â€” minimal stub: client name (h1), goal (subtext), "Coaching since" date, back link to `/clients`
2. Notes section displayed on the page (below client header)
3. Notes rendered as markdown when in view mode (bold, italic, lists, headings)
4. "Edit" button (or click empty state) switches to editable textarea (markdown input)
5. Notes auto-save on blur OR after 2-second debounce â€” no explicit save button needed
6. "Saving..." indicator during save; sonner toast "Notes saved" on success; toast on error
7. Notes persisted to `clients.notes TEXT` via `PUT /api/clients/[id]`
8. Notes survive page refresh (fetched via `GET /api/clients/[id]`)
9. Character limit: 10,000 chars with live counter displayed
10. Empty state: "Click to add notes about this client" â€” clicking enters edit mode
11. Mobile responsive
