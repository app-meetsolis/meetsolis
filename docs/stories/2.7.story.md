# Story 2.7: Manual Notes (Rich Text)

## Status
Not Started

## Story

**As a** executive coach,
**I want** to add and edit free-form notes on a client record,
**so that** I can capture context, observations, and information that doesn't fit into session transcripts.

## Acceptance Criteria

1. Client detail page includes a Notes section (below or alongside session timeline)
2. Notes displayed as rich text (markdown rendered — bold, italic, lists, headings)
3. "Edit Notes" button switches notes to editable textarea (markdown input)
4. Notes auto-save on blur or after 2-second debounce (no explicit save button required)
5. Save confirmation: subtle toast "Notes saved" on successful save
6. Notes persisted in `clients.notes TEXT` column
7. Notes survive page refresh (fetched from API)
8. Character limit: reasonable upper bound (10,000 characters), with counter displayed
9. Empty state: "Click to add notes about this client" (click → enters edit mode)
10. Mobile responsive: notes section usable on phone

## Dev Notes

- Notes are client-level, not session-specific. Sessions have their own transcript + summary fields.
- Use the existing `PUT /api/clients/[id]` route — notes is just another field in ClientUpdateSchema
- Consider using a simple textarea with markdown preview toggle rather than a heavy rich-text editor
