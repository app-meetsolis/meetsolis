# Story 4.1: Embeddings & Vector Index

## Status
Not Started

## Story

**As a** developer,
**I need** session embeddings stored in pgvector so Solis Intelligence can perform semantic search across a coach's session history,
**so that** Solis finds relevant sessions regardless of date.

## Acceptance Criteria

1. `CREATE EXTENSION IF NOT EXISTS vector` run in Supabase (verify already enabled or run)
2. `sessions.embedding vector(1536)` column exists (included in migration 015)
3. `CREATE INDEX USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100)` created
4. `apps/web/src/lib/ai/embeddings.ts` exports `generateEmbedding(text: string): Promise<number[]>`
5. Embedding auto-generated from `sessions.summary` text after Story 3.4 AI summarization
6. Embedding stored in `sessions.embedding` column
7. Provider-agnostic: uses same `AI_PROVIDER` env var as summarization
8. Placeholder provider returns 1536-dimension zero vector (no API cost)
9. OpenAI uses `text-embedding-3-small` model
10. Claude uses compatible embedding model (Voyage AI or OpenAI embedding endpoint)

## Dev Notes

- Supabase pgvector cosine distance operator: `<=>` (lower is more similar)
- For RPC: create a Postgres function `match_sessions(query_embedding vector, match_count int, filter_user_id text)` via Supabase SQL editor
- ivfflat index with `lists = 100` is appropriate for MVP scale (<10,000 sessions)
- Embedding cost: OpenAI text-embedding-3-small = $0.02/1M tokens. A 500-word summary ≈ 750 tokens = $0.000015 per session. Negligible.
