## Why Hybrid RAG (not pure semantic)

Coaches ask two types of questions:
- **Intent-based:** "Prepare me for next session" — semantic search wins (understands meaning)
- **Exact-term:** "What did John say about his CFO?" — keyword search wins (catches literal "CFO")

Pure vector search misses exact-term queries. Hybrid search (semantic + keyword, merged via RRF) gives
best-of-both-worlds retrieval with no new tools or services — Supabase supports both natively.

`tsvector` is auto-generated from existing columns (title + summary + key_topics) — zero application
code needed after the migration.

### Migration SQL

```sql
-- 020_hybrid_search.sql

-- 1. Keyword index on sessions
ALTER TABLE sessions
  ADD COLUMN IF NOT EXISTS fts tsvector
  GENERATED ALWAYS AS (
    to_tsvector('english',
      coalesce(title, '') || ' ' ||
      coalesce(summary, '') || ' ' ||
      coalesce(array_to_string(key_topics, ' '), '')
    )
  ) STORED;

CREATE INDEX IF NOT EXISTS idx_sessions_fts ON sessions USING GIN(fts);

-- 2. Hybrid search RPC (semantic + keyword, merged via RRF)
--    Falls back to semantic-only if p_query_text is blank.
--    Only searches sessions with status = 'complete' (have summary + embedding).
CREATE OR REPLACE FUNCTION hybrid_session_search(
  p_user_id         uuid,
  p_client_id       uuid,
  p_query_text      text,
  p_query_embedding vector(1536),
  p_match_count     int DEFAULT 3
)
RETURNS TABLE (
  id             uuid,
  title          text,
  session_date   date,
  summary        text,
  key_topics     text[],
  semantic_rank  float,
  keyword_rank   float
)
LANGUAGE sql STABLE
AS $$
  WITH semantic AS (
    SELECT id,
      ROW_NUMBER() OVER (ORDER BY embedding <=> p_query_embedding) AS rank
    FROM sessions
    WHERE user_id = p_user_id
      AND client_id = p_client_id
      AND status = 'complete'
      AND embedding IS NOT NULL
    LIMIT 20
  ),
  keyword AS (
    -- Skipped entirely when query text is blank: FULL OUTER JOIN below
    -- treats missing rows as 0 score (semantic-only fallback).
    SELECT id,
      ROW_NUMBER() OVER (
        ORDER BY ts_rank(fts, websearch_to_tsquery('english', p_query_text)) DESC
      ) AS rank
    FROM sessions
    WHERE user_id = p_user_id
      AND client_id = p_client_id
      AND status = 'complete'
      AND p_query_text <> ''
      AND fts @@ websearch_to_tsquery('english', p_query_text)
    LIMIT 20
  ),
  rrf AS (
    SELECT
      COALESCE(s.id, k.id) AS id,
      COALESCE(1.0 / (60 + s.rank), 0) +
      COALESCE(1.0 / (60 + k.rank), 0) AS rrf_score
    FROM semantic s
    FULL OUTER JOIN keyword k ON s.id = k.id
    ORDER BY rrf_score DESC
    LIMIT p_match_count
  )
  SELECT
    sess.id,
    sess.title,
    sess.session_date,
    sess.summary,
    sess.key_topics,
    COALESCE(1.0 / (60 + sem.rank), 0) AS semantic_rank,
    COALESCE(1.0 / (60 + kw.rank), 0)  AS keyword_rank
  FROM rrf
  JOIN sessions sess ON sess.id = rrf.id
  LEFT JOIN semantic sem ON sem.id = rrf.id
  LEFT JOIN keyword  kw  ON kw.id  = rrf.id
  ORDER BY rrf.rrf_score DESC;
$$;

GRANT EXECUTE ON FUNCTION hybrid_session_search TO authenticated, service_role;
```

**RRF note:** `1/(60 + rank)` is standard RRF formula (60 = smoothing constant). Sessions ranking high in
both semantic AND keyword search float to the top. Story 4.2 can use `semantic_rank`/`keyword_rank`
for transparency if needed.

## Risk Assessment

- **Empty query guard:** `p_query_text <> ''` condition in keyword CTE prevents `websearch_to_tsquery` from running on blank input — semantic-only fallback via FULL OUTER JOIN ✅ resolved
- **In-progress sessions:** `status = 'complete'` filter on both CTEs — sessions without summary/embedding never appear in results ✅ resolved
- **Zero-vector risk:** Mock/Claude providers return `Array(1536).fill(0)` — semantic CTE returns arbitrary ordering (all distances equal), but keyword CTE still works correctly
- **Migration:** additive only — new column + new function, no schema breaks
