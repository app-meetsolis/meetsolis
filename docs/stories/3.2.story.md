# Story 3.2: Manual Transcript Upload

## Status
Not Started

## Story

**As a** executive coach,
**I want** to upload my session transcript as a text file or docx, or paste it directly,
**so that** I can log a coaching session in under 2 minutes without any technical complexity.

## Acceptance Criteria

1. Accepted file types: `.txt` (direct text read) and `.docx` (parsed via mammoth to plain text)
2. Maximum file size: 25MB; error shown if exceeded
3. Paste/type text option: textarea supporting up to 50,000 characters with counter
4. Session date picker (required, defaults to today's date)
5. Session title input (required, e.g., "Session 5 — Leadership Transition")
6. Validation: at least one of (file upload OR pasted text) must be provided
7. Unsupported file type shows clear error message
8. File uploaded to Supabase Storage at `transcripts/[userId]/[sessionId]/[filename]`
9. Pasted text or extracted file text stored in `sessions.transcript_text`
10. File URL stored in `sessions.transcript_file_url`
11. On save: session created with `status = 'pending'`; AI summarization triggered automatically (POST to `/api/sessions/[id]/summarize`)
12. Success: modal closes, session card appears in timeline with "Processing..." state
13. Error states handled: file too large, wrong format, upload failed, network error

## Tasks / Subtasks

- [ ] **Transcript Parsing Utility**
  - [ ] Create `apps/web/src/lib/files/parseTranscript.ts`
  - [ ] `.txt`: read as UTF-8 string
  - [ ] `.docx`: `mammoth.extractRawText({ buffer })` → plain text
  - [ ] Returns: `{ text: string, sourceType: 'txt' | 'docx' | 'paste' }`
  - [ ] Install `mammoth` package if not already installed

- [ ] **Session Upload API**
  - [ ] `POST /api/sessions` accepts: `client_id`, `session_date`, `title`, `transcript_text` (optional), multipart file (optional)
  - [ ] Validate ownership: client belongs to authenticated user
  - [ ] Check transcript quota: `checkTranscriptLimit(userId)` — throws if limit exceeded
  - [ ] Upload file to Supabase Storage (if file provided)
  - [ ] Create session record
  - [ ] Increment transcript count: `incrementTranscriptCount(userId)`
  - [ ] Trigger summarization: `POST /api/sessions/[id]/summarize` (fire-and-forget)
  - [ ] Return: created session record (202 or 201)

- [ ] **SessionUploadModal — Tab 1: Manual Upload**
  - [ ] File input accepting `.txt` and `.docx` only
  - [ ] File size validation (client-side: reject >25MB)
  - [ ] OR: textarea for paste input with character counter (50,000 max)
  - [ ] Date picker (defaulting to today)
  - [ ] Title input (required)
  - [ ] Submit button: "Upload & Process"
  - [ ] Loading state during upload

- [ ] **Usage Limit Enforcement**
  - [ ] Check quota before upload
  - [ ] Show UpgradeModal if limit exceeded (free: 3 lifetime, pro: 25/month)

## Dev Notes

- `mammoth` is pure JS/TS, no native dependencies — safe for serverless
- Supabase Storage: use service role key for server-side uploads
- File upload: use `FormData` on the client; parse `multipart/form-data` on the API route with Next.js built-in parsing
- Summarization is triggered server-side after session created — client doesn't need to poll; session status updates can be polled via React Query refetch interval
