# Story 6.2c: Session Intelligence — Action Items Control, Transcript/Summary Views, Audio Playback

## Status
done

## Context

Continuation of Story 6.2b (real-time transcription). 6.2b Phase 1 + 2 shipped: bot joins, live streaming transcript, summary auto-generates on call end. This story adds the user-facing intelligence layer + fixes UX gaps found in live testing.

**Shipped (6.2b):** `LiveTranscriptPanel`, `/api/sessions/[id]/transcript-live`, `runSummarize` (currently generates summary + action items together), migration 026.

**Driver:** live-test feedback (2026-05-15) — transcript is buried, no summary read view, action items always auto-generate with no user control, dashboard meeting box doesn't scroll.

## Locked Decisions (2026-05-15)

- **Action items decoupled from summary.** AI service splits: `summarizeSession` returns summary + key_topics ONLY. New `generateActionItems(transcript, clientCtx)` returns action items. Pipeline becomes transcript → summary → action items (3 sequential, action items optional).
- **Action items are opt-in.** Per-session manual "Generate Action Items" button. Plus a global auto-toggle, **default OFF** — when on, action items auto-generate after every meeting's summary.
- **Toggle placement:** NOT in settings. Lives in the "Open Actions" panel header on the client detail page — easily accessible. Built as a self-contained `<ActionItemsAutoToggle />` so the upcoming client-page redesign can relocate it freely.
- **Audio playback: FREE — implement.** Recall already records the meeting (`video_mixed_mp4`). Audio streams direct from Recall's signed S3 URL to the browser — no new service, no metered cost, no Vercel bandwidth. Fetched on demand via Recall `GET /recording/{id}`.
- **Transcript viewing:** modal/drawer, opened by one button per session. Shows live transcript while streaming, clean final transcript after. No deep sub-page navigation.
- **Summary viewing:** read-only `<SummaryView />` (separate from the existing "Edit Summary" flow).
- **MODULARITY MANDATE:** client page + client detail page are being redesigned in upcoming stories. Every component here MUST be self-contained — clean props, no coupling to current page layout, no hardcoded positions. The redesign must be able to re-place these components without touching their internals.
