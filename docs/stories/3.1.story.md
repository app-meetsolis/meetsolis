## Acceptance Criteria

1. `sessions` table exists with all columns and indexes (pgvector enabled)
2. Deleting a client cascades to `sessions` and `action_items` via FK
3. Deleting a session cascades to its `action_items` via FK
4. `GET /api/sessions?client_id=[id]` returns sessions reverse-chronological, filtered by user
5. `POST /api/sessions` creates session — validates caller owns the client
6. `GET /api/sessions/[id]` returns session detail — ownership verified
7. `PUT /api/sessions/[id]` updates `title`, `summary`, `status`, `key_topics`
8. `DELETE /api/sessions/[id]` deletes session (cascade handled by DB)
9. `POST /api/sessions/[id]/summarize` returns 202 immediately (async — placeholder for Story 3.4)
10. Existing action-items routes continue to work unchanged
11. All routes return `{ error: { code, message } }` on error

## Database Schema (New Migration)

```sql
-- Enable pgvector if not already enabled
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  user_id TEXT NOT NULL,                   -- internal UUID (users.id)
  session_date DATE NOT NULL,
  title TEXT NOT NULL,
  transcript_text TEXT,
  transcript_file_url TEXT,
  transcript_audio_url TEXT,
  summary TEXT,
  key_topics TEXT[],
  embedding vector(1536),
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'processing', 'complete', 'error')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_client_sessions ON sessions(client_id);
CREATE INDEX idx_user_sessions ON sessions(user_id);
CREATE INDEX idx_session_date ON sessions(client_id, session_date DESC);
-- NOTE: ivfflat index requires data to exist first — create after first batch insert or defer
-- CREATE INDEX idx_sessions_embedding ON sessions USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "sessions_all" ON sessions FOR ALL USING (true);
-- Ownership enforced via user_id filter in API (same pattern as action_items)
```

**Migration note:** `action_items.session_id` FK already references `sessions(id)` — verify this FK exists. If not, add via `ALTER TABLE action_items ADD COLUMN IF NOT EXISTS session_id UUID REFERENCES sessions(id) ON DELETE CASCADE`.
