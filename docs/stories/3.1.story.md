# Story 3.1: Session DB Schema & API

## Status
Not Started

## Story

**As a** developer,
**I need** the sessions and action_items tables with RLS and API routes,
**so that** Epic 3 features can build on a solid data foundation.

## Acceptance Criteria

1. `sessions` table created with all specified columns and indexes
2. `action_items` table created with all specified columns and indexes
3. RLS enabled on both tables; users can only access their own data
4. `GET /api/sessions?client_id=[id]` returns sessions reverse-chronological for a client
5. `POST /api/sessions` creates a session (validates user owns the client)
6. `GET /api/sessions/[id]` returns session detail
7. `PUT /api/sessions/[id]` updates session fields (title, summary, status)
8. `DELETE /api/sessions/[id]` deletes session and cascades to action_items
9. `POST /api/sessions/[id]/summarize` triggers AI summarization (async)
10. `GET /api/action-items?client_id=[id]` returns action items for a client
11. `POST /api/action-items` creates a manual action item
12. `PUT /api/action-items/[id]` updates status, sets completed_at when completed
13. Cascade delete: deleting a client removes all its sessions and action_items (via DB FK)
14. Cascade delete: deleting a session removes its action_items (via DB FK)
15. All routes return standard error format `{ error: { code, message } }`

## Database Schema

```sql
CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  user_id TEXT NOT NULL,
  session_date DATE NOT NULL,
  title TEXT NOT NULL,
  transcript_text TEXT,
  transcript_file_url TEXT,
  transcript_audio_url TEXT,
  summary TEXT,
  key_topics TEXT[],
  embedding vector(1536),
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'processing', 'complete', 'error')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_client_sessions ON sessions(client_id);
CREATE INDEX idx_user_sessions ON sessions(user_id);
CREATE INDEX idx_session_date ON sessions(client_id, session_date DESC);
CREATE INDEX idx_sessions_embedding ON sessions USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

CREATE TABLE action_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
  client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  user_id TEXT NOT NULL,
  description TEXT NOT NULL,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'in_progress', 'completed', 'cancelled')),
  assigned_to TEXT CHECK (assigned_to IN ('coach', 'client')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);

CREATE INDEX idx_session_actions ON action_items(session_id);
CREATE INDEX idx_client_actions ON action_items(client_id);
CREATE INDEX idx_pending_actions ON action_items(user_id, status) WHERE status = 'pending';
```
