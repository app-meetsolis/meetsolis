# Story 3.3: Auto-Transcription

## Status
Not Started

## Story

**As a** executive coach,
**I want** to upload the audio or video recording of my coaching session and have it automatically transcribed,
**so that** I don't have to type or copy a transcript myself.

## Acceptance Criteria

1. Accepted formats: `.mp3`, `.mp4`, `.m4a`, `.wav`, `.webm`
2. Maximum file size: 500MB
3. File uploaded to Supabase Storage (`transcripts/[userId]/[sessionId]/audio/[filename]`)
4. URL stored in `sessions.transcript_audio_url`
5. Session `status` set to `'processing'` immediately after upload
6. Transcription job triggered via `TranscriptionProvider.transcribe(audioUrl)`
7. On success: `sessions.transcript_text` populated, `status` → `'complete'`, AI summarization triggered automatically
8. On failure: `status` → `'error'`; error visible in session card
9. Progress/processing indicator shown in session card while status is `'processing'`
10. Speaker diarization labels preserved in transcript when available (Deepgram only)
11. `TRANSCRIPTION_PROVIDER` env var controls which provider is used
12. Placeholder provider returns instant mock transcript (no API cost, for local dev)
13. Auto-transcript quota counted against AI session limit (same as manual + AI)

## Provider Abstraction

```typescript
// apps/web/src/lib/transcription/provider.ts
interface TranscriptionProvider {
  transcribe(audioUrl: string): Promise<TranscriptResult>
}

interface TranscriptResult {
  text: string
  speakers?: { label: string; text: string }[]
  duration?: number  // seconds
}
```

## Dev Notes

- Deepgram Nova-2 REST API: `POST https://api.deepgram.com/v1/listen?model=nova-2&diarize=true&punctuate=true` with audio URL in body
- Transcription is async server-side — the client polls session status (not a streaming response)
- For very long recordings, Deepgram async callback URL is preferred; for MVP, synchronous API call is acceptable (Node serverless timeout may be an issue for >10 min recordings — document limitation)
- Speaker diarization output: format as "Speaker A: [text]\nSpeaker B: [text]" when storing in transcript_text
