## Acceptance Criteria

1. Accepted formats: `.mp3`, `.mp4`, `.m4a`, `.wav`, `.webm`
2. Maximum file size: 500MB
3. File uploaded to Supabase Storage (`transcripts/[userId]/[sessionId]/audio/[filename]`)
4. URL stored in `sessions.transcript_audio_url`
5. Session `status` set to `'processing'` immediately after upload
6. Transcription job triggered via `getTranscriptionProvider().transcribe(audioUrl)`
7. On success: `sessions.transcript_text` populated, `status` → `'complete'`, `runSummarize()` triggered
8. On failure: `status` → `'error'`
9. Progress indicator shown in SessionCard while status is `'processing'`
10. Speaker diarization labels preserved: format as `"Speaker A: [text]\nSpeaker B: [text]"` in `transcript_text`
11. `TRANSCRIPTION_PROVIDER` env var controls which provider is used
12. Placeholder provider returns instant mock transcript (no API cost, for local dev)
13. Quota: call `checkTranscriptLimit()` before upload, `incrementTranscriptCount()` after success

## Dev Notes

- Deepgram sync API: `POST https://api.deepgram.com/v1/listen?model=nova-2&diarize=true&punctuate=true` with `Content-Type: application/json` body `{ "url": audioUrl }` and `Authorization: Token ${config.transcription.deepgramApiKey}`
- Deepgram response: `data.results.channels[0].alternatives[0].transcript` for full text; `data.results.utterances[]` for diarization (`{ speaker: 0|1, transcript: string }`)
- Speaker diarization format: `"Speaker A: {text}\nSpeaker B: {text}\n"` — store in `transcript_text`
- Sync API is fine for MVP; document known limitation: >10 min recordings may hit Vercel serverless timeout (10s default — consider increasing to 60s via `export const maxDuration = 60`)
- Reuse `uploadTranscriptFile()` from sessions API route — it accepts any file type
- The `runSummarize()` function in `summarize-session.ts` takes `(sessionId, userId)` — chain it after successful transcription
- `config.transcription.provider` already wired in `env.ts` — use that, not `process.env` directly
