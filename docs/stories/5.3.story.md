## Story

**As a** potential customer (executive coach),
**I want** a dedicated `/pricing` page that matches the homepage Novera-dark theme and answers my questions inline,
**so that** I can confidently decide to upgrade to Pro without leaving the site or feeling like I clicked into a different product.

## Context

**Already shipped (5.1, 5.6):**
- `/pricing` route exists at `apps/web/src/app/(marketing)/pricing/page.tsx` (3 tiers, monthly/annual toggle, Dodo checkout via `/api/billing/checkout`).
- Homepage uses `Novera*` dark-theme component family (`NoveraNavbar`, `NoveraPricing`, `NoveraFAQ`, `NoveraFooter`, etc.).
- `NoveraPricing` (414 lines) and `NoveraFAQ` (234 lines, 5 coach-specific Q&A) already exist and render the brand.

**Gaps blocking PRD AC:**
1. Existing `/pricing/page.tsx` is **722 lines, 100% inline styles** — violates `apps/web/CLAUDE.md` (≤500 lines, Tailwind only, no inline styles).
2. **No FAQ section** on `/pricing` — PRD 5.3 AC requires it.
3. Page does not reuse homepage chrome (`NoveraNavbar`, `NoveraFooter`) — inconsistent with `(marketing)` group.
4. Teams tier exists on page but not in PRD MVP scope (Free + Pro only) — needs decision.

**Approach:** Rebuild `/pricing/page.tsx` as a thin composition of existing `Novera*` components (mirrors `(home)/page.tsx` pattern). No new design work, no new visual components.
