## Key Behaviors

### Automatic Layout Switching
- **Screen share detected** â†’ Layout switches to Speaker mode
- **Screen share stops** â†’ Layout returns to previous mode (Gallery/Speaker)
- **Multiple participants** â†’ All video participants shown in filmstrip below

### Screen Share Priority
**Priority order:**
1. ğŸŸ¢ **Screen Share** (highest - always large)
2. Spotlight (host-assigned)
3. Pinned (user-pinned)
4. Active speaker
5. First participant

### Visual Indicators
- **Green button** = You are sharing
- **"User's Screen" label** = Shows whose screen is shared
- **Filmstrip** = Regular video participants (scrollable if >5)

### Mobile/Tablet Responsive
- **Filmstrip on mobile:** 2-3 visible thumbnails, horizontal scroll
- **Filmstrip on tablet:** 4-6 visible thumbnails
- **Desktop:** 6-8 visible thumbnails
- **Touch-friendly:** Large tap targets, swipe to scroll

## Technical Implementation

### Detection Logic
```typescript
// Screen share participants have sessionId ending with ':screen'
participant.sessionId.endsWith(':screen') // true for screen shares
```

### View Switching
- **SpeakerView**: Screen share â†’ main speaker
- **GalleryView**: Screen share â†’ switches to speaker-like layout
- **TwoPersonView**: Screen share â†’ replaces remote participant

### Stream SDK Rendering
- Uses `ParticipantView` component from Stream SDK
- Automatically renders screen share video track
- No manual track management needed
